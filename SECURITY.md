# Security Policy for W2UOS

This document defines the security policy and vulnerability disclosure process for the W2UOS project.

---

## 1. Supported Versions

Security updates are provided only for the latest stable version of W2UOS on the main branch.

Older snapshots, forks, and experimental branches are not guaranteed to receive security patches.

---

## 2. Reporting a Vulnerability

If you discover a security vulnerability in W2UOS:

- Do NOT open a public GitHub issue.
- Do NOT disclose the vulnerability publicly.
- Report the issue privately to the project author via GitHub.

The report should include:

- A clear description of the vulnerability
- Steps to reproduce
- Affected modules or components
- Potential impact and severity
- Suggested mitigation (if available)

---

## 3. Response Process

After receiving a vulnerability report:

1. The author will acknowledge receipt.
2. The issue will be investigated.
3. A patch or mitigation will be developed.
4. A security update will be released.
5. Public disclosure may occur after a fix is available.

Response time depends on the severity and complexity of the issue.

---

## 4. Security Scope

The security policy applies to:

- Core runtime and kernel
- Message bus and networking layers
- Execution engine and exchange adapters
- API and authentication system
- Logging and persistence layers
- Dashboard communication interfaces

User strategies, third-party plugins, and external exchange services are outside the scope of direct security guarantees.

---

## 5. No Financial Liability

W2UOS is provided as a technical software platform.

The author does NOT provide:

- Financial guarantees
- Trading performance guarantees
- Risk-free operation assurances

All financial risks resulting from the use of W2UOS are solely assumed by the user.

---

## 6. Responsible Disclosure

All vulnerability reports must follow responsible disclosure practices.

Public disclosure before an official fix is released is strictly prohibited.
